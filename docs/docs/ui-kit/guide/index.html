UI Kit Guide

This page is the AI-facing working guide for @zaisustech/ui-kit.

Read it in order:

  1. Installation
  2. Theme setup
  3. Styling conventions
  4. Component reference

The prop tables below are derived from the component source in libs/ui/ui-kit/src. They list the direct props declared by each component. When the Notes column names a parent type, the component also inherits that parent prop contract.

Phase 1 Installation​

Install the package plus the peer dependencies used by the exported components.

Package​

bun add @zaisustech/ui-kit
# or
npm install @zaisustech/ui-kit
# or
yarn add @zaisustech/ui-kit

Phase 2 Theme​

The theme contract lives in libs/ui/ui-kit/src/theme/createTheme.ts and libs/ui/ui-kit/src/theme/ThemeContext.tsx.

Theme contract​

FieldTypeSource
modelight | darkResolved by ThemeProvider
colorssemanticColors or semanticColorsDarkTheme color layer
spacingspacingToken scale
radiusradiusToken scale
shadowsReturnType<typeof globalShadows>Theme elevation layer
typographytypographyText styles
iconSizeiconSizeIcon sizing tokens
zIndexzIndexLayering tokens
animationanimationMotion timing and easing
opacityopacityTransparency tokens

Provider​

ThemeProvider accepts:

PropTypeNotes
childrenReact.ReactNodeRequired wrapper content
valueThemeOptional overrides merged into the resolved theme
mode'light' | 'dark' | 'system'Defaults to system

If value is supplied, it is merged with the resolved mode theme. If value is omitted, createTheme({ mode }) is used directly.

Setup​

import { ThemeProvider } from '@zaisustech/ui-kit';

function App() {
  return (
    <ThemeProvider mode="system">
      {/* app */}
    </ThemeProvider>
  );
}

Using the theme​

import { useTheme } from '@zaisustech/ui-kit';

function Example() {
  const { colors, spacing, radius, typography } = useTheme();

  return (
    <View
      style={{
        backgroundColor: colors.surface,
        padding: spacing.md,
        borderRadius: radius.md,
      }}
    >
      <Text style={typography.header.default.bold}>Hello</Text>
    </View>
  );
}

Phase 3 Styling Conventions​

Follow the codebase convention for every component-specific style hook:

  1. Create a use<Name>Style.ts file next to the component.
  2. Read from useTheme() instead of hard-coding colors, spacing, radius, or shadows.
  3. Use tokens from theme.colors, theme.spacing, theme.radius, theme.shadows, theme.typography, theme.iconSize, theme.zIndex, theme.animation, and theme.opacity.
  4. Keep style math close to the component that owns it.
  5. If a component needs a custom theme, pass it through ThemeProvider or createTheme, do not fork token names.

Example​

import { useTheme } from '@zaisustech/ui-kit';

export function useButtonStyle() {
  const { colors, radius, spacing, typography } = useTheme();

  return {
    container: {
      backgroundColor: colors.primary,
      borderRadius: radius.md,
      paddingHorizontal: spacing.lg,
      paddingVertical: spacing.sm,
    },
    title: {
      ...typography.label.bold,
      color: colors.secondaryText,
    },
  };
}

Component Rules​

  • Never invent props. Check the source or this table first.
  • Use the linked component doc page for usage examples and prop behavior.
  • When a component inherits another prop contract, include the inherited contract in your mental model before changing examples or styles.
  • Prefer token values over raw numbers and colors unless a component explicitly takes a raw value prop.

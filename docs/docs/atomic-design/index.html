Atomic Design Pattern

@zaisustech/ui-kit follows Atomic Design from the token layer upward.

Layers In This Library​

LayerWhat it containsSource of truth
TokensColor, spacing, radius, typography, shadow, animation, opacity, z-index, icon sizedocs/tokens.mdx and libs/ui/ui-kit/src/tokens
AtomsSmall, single-purpose building blockslibs/ui/ui-kit/src/atoms
MoleculesReusable combinations of atomslibs/ui/ui-kit/src/molecules
OrganismsFeature-sized UI surfaceslibs/ui/ui-kit/src/organisms
ThemeRuntime values consumed through useTheme()docs/theme.mdx

Public Component Map​

Atoms​

ActivityIndicator, Badge, Button, Divider, Icon, ProgressBar, ProgressCircle, Spacer, Switch, Text, TouchFeedbackButton, Visible

Molecules​

Accordion, BottomSheet, Card, Carousel, ColorPicker, EmptyState, FormFieldContainer, Header, ListItem, PageIndicator, Pager, RangeSlider, RichText, SegmentedControl, Slider, Snackbar, Stepper, TextInputField, Toast

Organisms​

ActionSheet, CalendarView, ClockView, DatePicker, Dialog, DropdownField, ScrollPicker, TabView

How To Use This Pattern​

  1. Start with tokens instead of hard-coded values.
  2. Build leaf UI with atoms.
  3. Compose atoms into molecules.
  4. Compose molecules into organisms.
  5. Keep styling in a sibling use<Name>Style.ts file so the component stays readable.

Where The Prop Details Live​

  • Use the UI Kit Guide for the source-derived component prop inventory.
  • Use the individual component docs under UI-Kit components for examples and behavior.

Practical Rule​

When adding a new component or editing an existing one, do not invent new token names or props. Check the source types first, then update the docs so the docs match the code.
